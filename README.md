# Aufgabe 0 Vorbereitung

## Softwareprodukt

Fuer mein "Fake" Start-up entwickle ich einen Webshop fuer **bestauth**. 

Bestauth ist eine Authenticator-App zur Verwaltung von 2FA-Authentifizierungscodes (Anwendungsfall z.B. TU Wien 2FA). Die Anwendung wird Nutzerinnen helfen Login-Codes einfach und sicher zu verwalten mit dem Alleinstellungsmerkmal das bestauth auf dem Smartphone als auch am Desktop verwendet werden kann (Mit dem selben Account).

## Funktionen der Anwendung

bestauth bietet folgende Funktionen:

- Verwaltung von 2FA-Authentifizierungscodes
- Uebersichtliche Anzeige von "Time-based One-Time Password"
- Nutzung auf Smartphone und Desktop
- Synchronisierung zwischen mehreren Geraeten
- Backup-Funktion
- lokale App-Sperre mit PIN
- Dark Mode
- WCAG 2.2 Konformes UI

Die Website beschreibt nur die Funktionen aber die Applikation wird im Rahmen der Uebung nicht erstellt sondern nur als "Idee" dargestellt.

## Zielgruppe

Zielgruppe von bestauth sind Nutzerinnen die ihre Konten mit 2FA einfacher verwalten wollen.

Relevante Gruppen zum Beispiel:

- Studierende
- Entwicklerinnen und Entwickler
- Selbststaendige Unternehmer oder kleine Teams

## Unternehmen

Das Unternehmen des Produktes ist **bestauth Software e.U.** und der Standort des Unternehmens ist Wien, Oesterreich. 

Es handelt sich um ein kleines Software-Start-up mit **4 Mitarbeiterinnen** und der Fokus des Unternehmens liegt auf einfacher Funktion, Datenschutz und benutzerfreundlicher Gestaltung.
